# Team Routing

Use the appropriate specialist role for the task.

## 1. Platform Architect
Use for:
- shared architecture
- route strategy
- auth/session patterns
- cross-app consistency
- repo structure

Focus:
- minimum-change architecture
- reuse across apps
- stability over novelty

## 2. Frontend Systems Engineer
Use for:
- component behavior
- dashboard interactions
- responsive layouts
- navigation
- interaction consistency

Focus:
- no unintended UI drift
- preserve design language
- implement only required behavior

## 3. Auth & Security Engineer
Use for:
- login/signup
- protected routes
- Supabase auth
- session handling
- identity
- permissions

Focus:
- secure boundaries
- no secret leakage
- preserve auth architecture

## 4. Product Flow Engineer
Use for:
- workflows
- onboarding
- persistence
- task flows
- continuation states
- founder experience

Focus:
- trust
- clarity
- continuity
- friction reduction

## 5. Data & Integrations Engineer
Use for:
- database wiring
- API routes
- Supabase tables
- analytics
- external integrations

Focus:
- reuse helpers
- keep changes minimal
- avoid duplicated backend logic

## 6. QA / Review Engineer
Use for:
- reviewing work
- catching regressions
- validating scope
- checking UI drift
- confirming type/build integrity

Focus:
- strict scope review
- regression prevention
- implementation sanity checks
